# Weiyinblue v1.0.0 Release Draft

## Summary

`@weiyinblue/ui` reaches `v1.0.0` as a stable, GitHub-ready design-system package for:

- portfolio and demo sites
- campus and settings-heavy forms
- admin and dashboard surfaces

This release freezes the documented package contract and moves the repository into maintenance mode.

## Stable scope

The `v1.0.0` contract covers:

- theme entry points: `ThemeProvider`, `ThemeToggle`, `styles.css`
- documented primitives
- documented form helpers
- documented blocks
- starter templates and docs examples that consume public exports only

## Supported use cases

- reusable page shells for portfolio landing pages
- form-heavy application and settings flows
- dashboard pages with metrics, filters, tables, loading states, and empty states

## Non-goals

- broad component-catalog expansion
- enterprise-grade data-grid behavior
- charts, calendars, or command systems
- npm distribution as part of the `v1.0` milestone

## Validation status

Release gate:

- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
- `pnpm build`
- `pnpm test:e2e`

Supporting verification:

- docs site uses public exports only
- `apps/consumer-smoke` verifies external-consumer composition inside the monorepo
- visual/e2e protection covers theme and example surfaces

## Maintenance policy

After `v1.0.0`, default changes should be limited to:

- bug fixes
- accessibility fixes
- React / Next.js / Tailwind / PostCSS compatibility
- repeated high-frequency additions proven across real usage
